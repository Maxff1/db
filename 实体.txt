# 校运会管理系统 - 实体清单

## 用户
- **说明**：系统用户，通过角色区分权限
- **核心属性**：用户ID，用户名，密码，角色，姓名，部门（可空）
- **角色枚举**：管理员 / 校体育处 / 裁判 / 运动员 / 观众
- **外键**：无
- **对应操作**：创建、删除、分配角色（由管理员操作）

## 赛事
- **说明**：运动会赛事
- **核心属性**：赛事ID，名称，描述，开始日期，结束日期，状态
- **外键**：创建人 → 用户
- **对应操作**：创建、编辑、删除、查看；管理员可维护任意赛事

## 项目
- **说明**：赛事下的具体比赛项目
- **核心属性**：项目ID，名称，类型，性别限制，最少人数，最多人数，最大报名队伍数
- **外键**：赛事ID → 赛事
- **类型枚举**：个人 / 团体
- **性别限制枚举**：男 / 女 / 混合
- **对应操作**：添加项目、配置性别/人数/名额

## 场地
- **说明**：比赛场地，通过赛程占用，状态可由赛程时间自动判定
- **核心属性**：场地ID，名称，状态
- **状态枚举**：空闲 / 使用中（应用层根据当前时间的赛程自动维护或定时更新）
- **外键**：无
- **对应操作**：通过为赛程分配场地实现“借用”，赛程结束后自动“归还”

## 赛程
- **说明**：单场比赛排期，决定比赛时间、地点和阶段
- **核心属性**：赛程ID，轮次，开始时间，结束时间，状态
- **外键**：项目ID → 项目，场地ID → 场地
- **轮次示例**：预赛 / 半决赛 / 决赛
- **状态枚举**：已安排 / 已发布 / 已完成
- **对应操作**：分配场地、分配时段、分配裁判、发布赛程
- **约束**：同一场地同一时段不能存在两个状态为“已安排/已发布”的赛程

## 执裁安排
- **说明**：裁判执裁安排（多对多关系）
- **核心属性**：赛程ID，裁判ID（两个字段联合作为主键）
- **外键**：赛程ID → 赛程，裁判ID → 用户
- **对应操作**：为赛程分配裁判；裁判查看本人执裁场次

## 参赛安排
- **说明**：赛程的参赛安排，决定哪场比赛谁上场
- **核心属性**：参赛安排ID，参与类型
- **外键**：赛程ID → 赛程，场地→ 场地，参与方ID→ 运动员/队伍
- **参与类型枚举**：运动员 / 队伍
- **对应操作**：安排运动员/队伍上场，分配场地

## 成绩
- **说明**：某场比赛某参赛者的最终成绩
- **核心属性**：成绩ID，成绩详情，名次，状态
- **外键**：参赛安排ID → 参赛安排，录入人 → 用户（裁判），审核人 → 用户（校体育处）
- **状态枚举**：已提交 / 已审核 / 已发布
- **对应操作**：裁判录入提交、体育处审核发布、修正已发布成绩

## 队伍
- **说明**：运动员队伍，需经体育处审核通过方可生效
- **核心属性**：队伍ID，名称，状态
- **外键**：队长ID → 用户
- **状态枚举**：待审核 / 已通过 / 已拒绝 / 已解散
- **对应操作**：队长创建队伍（提交组队申请）、体育处审核组队申请、队长解散队伍

## 队伍成员关系
- **说明**：队伍成员关系
- **核心属性**：队伍ID，运动员ID，状态
- **外键**：队伍ID → 队伍，运动员ID → 用户
- **状态枚举**：已邀请 / 已加入 / 已移除
- **对应操作**：邀请队员、移除队员、接受/拒绝邀请

## 报名记录
- **说明**：报名记录
- **核心属性**：报名ID，状态，申请时间
- **外键**：项目ID → 项目，运动员ID → 用户（个人，可为空），队伍ID → 队伍（团体，可为空）
- **状态枚举**：待审核 / 已通过 / 已拒绝
- **对应操作**：个人报名、队伍报名；自动校验名额/性别；体育处审核

## 公告
- **说明**：系统公告或赛事通知
- **核心属性**：公告ID，标题，内容，创建时间，状态
- **外键**：创建人 → 用户，赛事ID → 赛事（可为空）
- **状态枚举**：正常 / 已删除
- **对应操作**：发布系统级公告、发布赛事通知、删除不当公告

## 操作日志
- **说明**：关键操作日志
- **核心属性**：日志ID，动作，描述，时间戳
- **外键**：用户ID → 用户
- **对应操作**：记录用户管理、成绩修改、审核等关键动作；管理员查看
