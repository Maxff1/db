# 数据库课程设计 —— 完整 Git 开发流程

一、分支说明
========================================
- main      最终交作业版本（最后才用）
- dev       日常开发共用版本（所有人合并到这里）
- 姓名-功能  个人开发分支（例：zhang-student）

二、开发流程（从头到尾完整版）
========================================


【第一步】第一次拉项目（每个人只做一次）
----------------------------------------
git clone 仓库地址
cd 项目文件夹

【第二步】开始做一个新功能
----------------------------------------
1. 切换到 dev 分支
git checkout dev

2. 拉取最新代码（必须做！）
git pull

3. 创建自己的功能分支
git checkout -b zhang-student

【第三步】写代码 + 提交 + 上传
----------------------------------------
git add .
git commit -m "add: 完成学生查询功能"
git push origin zhang-student

【第四步】去 GitHub 提交合并申请（PR）
----------------------------------------
1. 打开仓库
2. 点击 Pull Request
3. 源分支：你的分支（zhang-student）
4. 目标分支：dev
5. 提交，等待组长同意合并

【第五步】功能没做完，第二天继续写
----------------------------------------
# 第二天打开电脑，继续同一个功能，不用建新分支！

1. 切回自己的分支
git checkout zhang-student

2. 同步 dev 的最新代码
git pull origin dev

3. 继续写代码 → 提交 → 上传
git add .
git commit -m "add: 完成学生新增功能"
git push origin zhang-student

【第六步】功能全部做完，合并到 dev
----------------------------------------
去 GitHub 提交 PR → 组长审核通过 → 合并成功

【第七步】做下一个新功能
----------------------------------------
# 先回到 dev 更新代码
git checkout dev
git pull

# 创建新分支
git checkout -b zhang-course

# 继续开发……

三、最重要的规则（必看）
========================================
1. 一个功能 = 一个分支，不用每天新建
2. 永远不要直接往 main / dev 推送代码
3. 每天开工前必须 git pull
4. 合并必须在 GitHub 网页提交 PR
